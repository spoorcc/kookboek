# Kookboek — GitHub Pages

A searchable recipe index with an in-browser PDF viewer, auto-deployed on every push to `main`.

## One-time setup

### 1. Enable GitHub Pages

Go to **Settings → Pages** in the repository and set:

| Field | Value |
|---|---|
| Source | Deploy from a branch |
| Branch | `gh-pages` |
| Folder | `/ (root)` |

Save. The site will be live at `https://<owner>.github.io/<repo>/` after the first successful workflow run.

### 2. Merge this branch to `main`

The workflow (`.github/workflows/gh-pages.yml`) only runs on pushes to `main`. Once merged, it will:

1. Build the PDF with XeLaTeX
2. Extract recipe titles, chapters, and page ranges from the PDF bookmarks (`scripts/extract_index.py`)
3. Copy `KookboekFamilieSpoor.pdf` and the generated `recipes.json` into `docs/`
4. Push the `docs/` folder to the `gh-pages` branch

## How the site works

| File | Role |
|---|---|
| `docs/index.html` | Single-page app — committed to the repo |
| `docs/recipes.json` | Recipe index — generated by CI, not committed |
| `docs/KookboekFamilieSpoor.pdf` | Built PDF — generated by CI, not committed |

On load, `index.html` fetches `recipes.json` to build the sidebar, then fetches the PDF once and caches it in memory (shared between the PDF.js viewer and pdf-lib for downloads).

## Features

- **Search** — filters the recipe list in real time by title
- **Two-page spread** — renders the recipe's PDF pages side by side on wide screens, single page on mobile
- **↓ Recept** — downloads only the current recipe's pages as a standalone PDF (uses pdf-lib in the browser, no server needed)
- **↓ Volledig boek** — downloads the complete cookbook PDF

## Local preview

You need a local HTTP server (browsers block `fetch()` from `file://`).

```sh
# From the repo root — after a local LaTeX build that produced KookboekFamilieSpoor.pdf:
python3 scripts/extract_index.py
cp KookboekFamilieSpoor.pdf docs/
cd docs && python3 -m http.server 8080
```

Then open `http://localhost:8080`.
